# Диздок: Библиотека книг (Library Service)

## 1. Описание проекта

Простой REST-сервис на Java + Spring Boot для управления небольшой
библиотекой: каталог книг с авторами, читатели и владения книгами.
Сервис предоставляет HTTP API в формате JSON; данные хранятся в
in-memory базе H2 через Spring Data JPA. Учебный проект, цель —
показать связку Controller → Service → Repository, базовую работу
с JPA-связями и обработку ошибок.

## 2. Стек

- **Язык:** Java 21
- **Сборка:** Maven
- **Фреймворк:** Spring Boot 3.4.x
- **БД:** H2(чтобы не требовать установки СУБД)
- **Тесты:** JUnit 5, Spring Boot Test, MockMvc

## 3. Доменная модель

Пять сущностей:

| Сущность | Поля | Связи |
|----------|------|-------|
| `Author` | `id`, `firstName`, `lastName` | one-to-many → `Book` |
| `Book`   | `id`, `title`, `description`, `year`, `pageCount` | many-to-one → `Author`; one-to-many → `Loan` |
| `Reader` | `id`, `firstName`, `lastName`, `email` | one-to-many → `Loan`, `ReadingProgress` |
| `Loan`   | `id`, `loanDate`, `returnDate` (nullable) | many-to-one → `Book`, `Reader` |
| `ReadingProgress` | `id`, `currentPage`, `updatedAt` | many-to-one → `Reader`, `Book`; уникальная пара `(reader, book)` |

Книга считается **доступной** всегда: владение не блокирует других читателей.

Прогресс чтения хранится отдельно от выдач: даже когда читатель вернул
книгу, его закладка сохраняется и при следующей выдаче он продолжит
с той же страницы. Сами файлы книг сервис **не хранит** — он работает
как трекер закладок поверх внешних источников (физическая книга,
e-reader и т.п.).

## 4. Бизнес-требования(инварианты)

1. Нельзя удалить автора, у которого есть книги.
2. Нельзя удалить книгу, которая сейчас выдана.
3. Владение одной книгой может одновременно принадлежать нескольким читателям.
4. Возврат владения не предусмотрен.
5. Email читателя должен быть валидным и уникальным.
6. Год издания книги — положительное число, не больше текущего года.
7. `pageCount` книги — положительное число.
8. `currentPage` в прогрессе — целое в диапазоне `[0, book.pageCount]`.
9. На пару (читатель, книга) может существовать только одна запись `ReadingProgress`.

## 5. REST API

Базовый префикс: `/api`. Все запросы/ответы — JSON.

### Авторы

| Метод | Путь                | Описание                  |
|-------|---------------------|---------------------------|
| GET   | `/authors`          | Список всех авторов       |
| GET   | `/authors/{id}`     | Один автор                |
| POST  | `/authors`          | Создать автора            |
| PUT   | `/authors/{id}`     | Обновить автора           |
| DELETE| `/authors/{id}`     | Удалить автора            |

### Книги

| Метод | Путь                          | Описание                        |
|-------|-------------------------------|---------------------------------|
| GET   | `/books`                      | Список книг (фильтр `?authorId=`) |
| GET   | `/books/{id}`                 | Одна книга                      |
| POST  | `/books`                      | Создать книгу                   |
| PUT   | `/books/{id}`                 | Обновить книгу                  |
| DELETE| `/books/{id}`                 | Удалить книгу                   |

### Читатели

| Метод | Путь                | Описание                     |
|-------|---------------------|------------------------------|
| GET   | `/readers`          | Список читателей             |
| GET   | `/readers/{id}`     | Один читатель                |
| POST  | `/readers`          | Зарегистрировать читателя    |
| PUT   | `/readers/{id}`     | Обновить данные читателя     |
| DELETE| `/readers/{id}`     | Удалить читателя             |

### Выдачи

| Метод | Путь                          | Описание                                    |
|-------|-------------------------------|---------------------------------------------|
| GET   | `/loans`                      | Список владений (фильтр `?readerId=`) |
| POST  | `/loans`                      | Выдать книгу: `{ "bookId": .., "readerId": .. }` |
| POST  | `/loans/{id}/return`          | Легаси-эндпоинт; возврат не используется     |

### Прогресс чтения

| Метод | Путь                                              | Описание                                            |
|-------|---------------------------------------------------|-----------------------------------------------------|
| GET   | `/readers/{readerId}/progress`                    | Все закладки читателя (по всем книгам)              |
| GET   | `/readers/{readerId}/progress/{bookId}`           | Закладка по конкретной книге                        |
| PUT   | `/readers/{readerId}/progress/{bookId}`           | Создать/обновить закладку: `{ "currentPage": 9 }`   |
| DELETE| `/readers/{readerId}/progress/{bookId}`           | Удалить закладку                                    |

## 7. Обработка ошибок
Станартная:
`GlobalExceptionHandler` (`@RestControllerAdvice`) преобразует исключения в HTTP-ответы:

- `NotFoundException` → 404
- `ConflictException` (нарушение бизнес-правил) → 409
- `MethodArgumentNotValidException` (ошибки валидации) → 400

Примерный формат ответа об ошибке:

```json
{ "status": 404, "error": "Not Found", "message": "Author 5 not found" }
```
